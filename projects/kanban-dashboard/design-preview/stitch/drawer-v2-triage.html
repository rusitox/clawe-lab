Login throws 500 on Safari 17

BUG P0
Marian
block Blocked by CKB-12 (Release 1.4.0)

Description

When users attempt to log in using Safari version 17.x, the authentication endpoint returns a 500 Internal Server Error. Other browsers (Chrome, Firefox) are unaffected.

Reproduction Steps

  1. Open Safari 17 or later
  2. Navigate to the login page at /auth/login
  3. Submit form with valid credentials
  4. Observe the 500 error in the network panel

Initial logs suggest a failure in processing the User-Agent header or a cookie serialization issue specific to the WebKit engine.
